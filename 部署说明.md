## 三、启动项目

### 1. 启动后端服务
```bash
cd api
node server.js
```
后端服务会自动从40000端口开始寻找可用端口，启动成功后会将端口号写入项目根目录的 `.api_port` 文件中。

### 2. 启动前端服务
```bash
# 在项目根目录
node node_modules/vite/bin/vite.js --host
```
前端默认运行在 http://localhost:5173

## 六、权限隔离机制

1. **登录分离**：三个角色使用独立的登录页面和登录接口
2. **Token分离**：
   - 学生用户：`localStorage.token` + `localStorage.user`
   - 超级管理员：`localStorage.super_admin_token` + `localStorage.super_admin`
   - 业务员：`localStorage.agent_token` + `localStorage.agent`
3. **接口鉴权**：
   - 管理员接口使用 `adminAuth` 中间件
   - 超级管理员接口使用 `superAdminAuth` 中间件
   - 业务员接口使用 `agentAuth` 中间件
4. **数据隔离**：
   - 业务员只能看到分配给自己的订单
   - 业务员看不到平台总收入、其他业务员信息、定价设置等
   - 超级管理员拥有全部权限

## 七、如何新增业务员账号

### 方法一：通过超级管理员后台（推荐）
1. 使用超级管理员账号登录 `/super-admin/login`
2. 进入"业务员管理"页面
3. 点击"新增业务员"按钮
4. 填写信息：
   - 登录账号（如 agent002）
   - 姓名
   - 手机号
   - 登录密码
   - 勾选可接业务（可多选：打印/网课/快递/代写）
5. 点击确定保存

### 方法二：直接修改数据库
1. 打开 `data/agents.json` 文件
2. 参考已有格式添加新业务员对象：
```json
{
  "id": 2,
  "username": "agent002",
  "password": "bcrypt加密后的密码",
  "name": "李师傅",
  "phone": "13700137002",
  "avatar": "",
  "services": ["print", "express"],
  "status": 1,
  "total_orders": 0,
  "total_income": 0,
  "created_at": "2026-01-01T00:00:00.000Z"
}
```
3. 注意：password 字段需要使用 bcrypt 加密

## 八、如何重置超级管理员密码

如果忘记超级管理员密码，可以直接修改数据库：

1. 打开 `data/users.json` 文件
2. 找到 `role: "super_admin"` 的用户
3. 修改 password 字段为新的 bcrypt 加密密码

或者使用以下方式生成新密码：
```bash
cd api
node -e "const bcrypt = require('bcryptjs'); console.log(bcrypt.hashSync('新密码', 10))"
```
将生成的哈希值替换到 users.json 中即可。

### 订单状态说明
| 状态 | 说明 |
|------|------|
| `pending` | 待支付 |
| `paid` | 已支付（待分配） |
| `assigned` | 已分配（待接单） |
| `accepted` | 已接单（处理中） |
| `processing` | 处理中 |
| `completed` | 已完成 |
| `cancelled` | 已取消 |
| `refunded` | 已退款 |

## 十、订单流转流程

1. 学生下单 → 订单状态：pending（待支付）
2. 学生支付 → 订单状态：paid（已支付，进入订单池）
3. 超级管理员手动分配订单给业务员 → 订单状态：assigned（已分配）
   - 或者业务员在订单池抢单
4. 业务员接单 → 订单状态：accepted / processing（处理中）
5. 业务员完成订单，上传凭证 → 订单状态：completed（已完成）
6. 订单金额计入业务员收入

## 十一、常见问题

### 1. 前端页面加载失败
- 检查后端服务是否启动
- 检查 `.api_port` 文件中的端口号是否与后端实际端口一致
- 重启前端开发服务器

### 2. 登录提示"账号或密码错误"
- 确认账号密码是否正确
- 检查是否使用了对应角色的登录页面
- 查看 data/users.json 或 data/agents.json 中的账号状态是否为1（正常）

### 3. 业务员看不到订单
- 检查该业务员是否有对应服务类型的接单权限
- 检查订单是否已分配给该业务员
- 检查订单是否在订单池中（已支付未分配）
